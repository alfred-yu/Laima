## 4. 技术实现细节

### 角色定义
- **普通用户** (`user`)：只能看到个人相关内容
- **管理员** (`admin`)：可以看到系统级内容和管理功能

### 状态管理增强
```typescript
// 在 useUserStore 中添加
state: () => ({
  user: null as any,
  token: localStorage.getItem('token') || '',
  role: localStorage.getItem('role') || 'user', // 默认普通用户
  // 其他状态...
}),
getters: {
  isLoggedIn: (state) => !!state.token,
  currentUser: (state) => state.user,
  isAdmin: (state) => state.role === 'admin',
  isUser: (state) => state.role === 'user'
},
actions: {
  // 其他 actions...
  setRole(role: string) {
    this.role = role
    localStorage.setItem('role', role)
  }
}
```

### 仪表盘条件渲染
```vue
<!-- 普通用户内容 -->
<div v-if="!isAdmin" class="user-dashboard">
  <!-- 个人相关内容 -->
</div>

<!-- 管理员内容 -->
<div v-else class="admin-dashboard">
  <!-- 系统管理相关内容 -->
</div>
```

## 5. 依赖与考虑

### 依赖
- **Pinia**：用于状态管理
- **Vue Router**：用于路由导航
- **Vue 3**：使用 Composition API

### 考虑事项
- **安全性**：确保普通用户无法访问管理员功能
- **用户体验**：为不同角色提供合适的内容和操作
- **可扩展性**：便于未来添加更多角色类型
- **性能**：条件渲染不影响页面加载性能

## 6. 风险处理

### 潜在风险
- **状态同步**：角色信息可能与实际用户权限不一致
- **缓存问题**：localStorage 中的角色信息可能过期
- **权限绕过**：前端角色控制可能被绕过

### 解决方案
- **后端验证**：所有敏感操作必须在后端进行权限验证
- **定期同步**：定期从后端同步用户角色信息
- **前端保护**：前端路由守卫和权限检查作为辅助手段
